# AI Agent Portfolio Project Plan

## 1. プロジェクト概要

このプロジェクトは、LLM APIを利用してユーザーの入力を理解し、必要に応じて外部ツールや処理を実行できるAIエージェントを開発するポートフォリオです。

まずはローカル環境で最小構成のAIエージェントを作成し、その後、必要に応じてWeb化・データ保存・クラウドデプロイ・監視・停止機能などを追加していきます。

## 2. 目的

このプロジェクトの目的は、以下の開発力を示すことです。

- OpenAI APIを使ったLLMアプリケーション開発
- AIエージェントの基本設計
- 外部APIやツール呼び出しの実装
- ログ・エラー処理・安全停止を考慮した設計
- 将来的なクラウド運用を見据えた構成
- READMEや設計ドキュメントを含めた実務的な開発管理

## 3. 想定ユーザー

- 採用担当者
- エンジニア面接官
- ポートフォリオを見る第三者
- 開発者本人

## 4. 現在の開発方針

現時点では、先にAWSなどのインフラを構築するのではなく、ローカル環境でAIエージェントの中核機能を固める。

理由:

- エージェントの仕様がまだ変わる可能性が高い
- 先にクラウド化すると課金・停止・削除管理が複雑になる
- ポートフォリオとしては、まず動く中核機能を作ることが重要
- ローカルで設計を固めた後の方が、クラウド構成を整理しやすい

## 5. MVP

最初に作る最小構成は以下とする。

- OpenAI APIに接続できる
- ユーザーの入力を受け取れる
- AIからの応答を表示できる
- 簡単なツール呼び出しを1〜2個実装する
- エラー時に分かりやすいメッセージを出す
- APIキーを `.env` で管理する
- 実行手順をREADMEにまとめる

## 6. 現在の進捗

- Pythonプロジェクトを作成済み
- 仮想環境を作成済み
- `.env` にOpenAI APIキーを設定済み
- OpenAI APIへの接続確認済み
- `requirements.txt` を作成済み
- Git管理を開始済み
- 初回コミット前の状態

## 7. 今後追加したい機能

### ローカル開発フェーズ

- CLIでユーザー入力を受け取る
- 複数ターンの会話に対応する
- 会話履歴を保持する
- エラー処理を整理する
- ログ出力を追加する
- コードを複数ファイルに分割する

### エージェント機能フェーズ

- ツール呼び出し機能を追加する
- 簡単な外部APIと連携する
- ユーザー入力に応じて処理を分岐する
- 実行結果をAIの応答に反映する
- 危険な処理を勝手に実行しない設計にする

### Web化フェーズ

- FastAPIなどでAPIサーバー化する
- フロントエンドからチャットできるようにする
- APIキーをフロントエンドに出さない構成にする
- CORSや環境変数の管理を整理する

### 保存・管理フェーズ

- 会話ログを保存する
- 実行ログを保存する
- エラー履歴を確認できるようにする
- 管理者用の停止機能を検討する

### クラウド化フェーズ

- AWS上にデプロイする
- 課金要素を整理する
- 停止手順を用意する
- 削除手順を用意する
- コストアラートを設定する
- 必要であれば監視・通知を追加する

## 8. 技術スタック候補

現時点の候補。

- 言語: Python
- LLM API: OpenAI API
- 環境変数管理: python-dotenv
- Web API: FastAPI
- フロントエンド: 未定
- データベース: SQLite または PostgreSQL
- インフラ: AWS
- バージョン管理: Git / GitHub

## 9. 設計上の注意点

### APIキー管理

- `.env` にAPIキーを保存する
- `.env` はGitHubに上げない
- `.env.example` を用意する
- フロントエンドにAPIキーを置かない

### コスト管理

- API呼び出し回数を意識する
- 無限ループでAPIを叩かない
- 将来的に利用量のログを残す
- AWS利用時は停止・削除手順を用意する
- コストアラートを設定する

### セキュリティ

- ユーザー入力をそのまま危険な処理に渡さない
- 外部ツール実行には制限を設ける
- 管理者用機能には認証をつける
- ログにAPIキーや個人情報を出さない

### エラー処理

- APIキー未設定時に分かりやすく失敗する
- API接続エラーを捕捉する
- タイムアウトやレート制限を考慮する
- ユーザー向けメッセージと開発者向けログを分ける

## 10. ポートフォリオとして見せたいポイント

- 単にAPIを叩くだけではなく、設計意図を説明できること
- ローカル開発からクラウド運用まで段階的に考えていること
- APIキーや課金管理など、実務的な注意点を理解していること
- エージェントの暴走や危険操作を防ぐ設計を考慮していること
- README、TODO、設計メモを残しながら開発していること

## 11. 現時点でやらないこと

初期段階では、以下は優先しない。

- 本格的な認証機能
- 複雑な管理画面
- マルチユーザー対応
- 高度なRAG
- 本番レベルの監視基盤
- 複雑なAWS構成
- 自動課金停止システム
- スマホ対応の管理画面

まずはAIエージェントとしての中核機能を完成させる。
